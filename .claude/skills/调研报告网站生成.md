# Skill: 调研报告网站生成 v1.0

> 基于「世界模型研究报告」VitePress 网站项目的实战经验提炼。将单文件深度调研报告拆分为结构化多页网站，部署到 GitHub Pages。

## 目标

将一份**单文件深度调研报告**（由「深度行业调研」skill 产出）转化为一个**高质量 VitePress 静态网站**，具备搜索、导航、可排序表格、KaTeX 公式渲染、GitHub Pages 自动部署等能力。

## 适用场景

| 场景 | 说明 |
|------|------|
| 调研报告发布 | 将 Markdown 报告转化为可在线阅读的网站 |
| 技术文档站 | 适合多章节、多方向的技术文档 |
| 学术综述展示 | 论文链接密集、表格多、公式多的内容 |

## 设计原则

| 原则 | 说明 |
|------|------|
| 内容驱动 | 文件结构直接映射报告章节，无冗余路由配置 |
| 中文优先 | 全站中文 UI，搜索支持中文分词 |
| 学术友好 | KaTeX 公式、arXiv/GitHub 链接样式、可排序数据表格 |
| 一键部署 | GitHub Actions 自动构建 + 部署，push 即上线 |
| 增量维护 | 编辑单个 Markdown 文件即可更新网站，无需触碰框架代码 |

```
第一轮：项目初始化（脚手架 + 依赖 + CI/CD）
```

## 第二轮：内容拆分

### 拆分原则

| 原则 | 说明 |
|------|------|
| **一章一文件** | 报告中的每个 `## 章节` 对应一个独立 `.md` 文件 |
| **目录映射** | 文件路径直接映射侧边栏结构 |
| **frontmatter** | 每个文件顶部加 `outline: deep` 以启用右侧目录 |
| **标题降级** | 原报告的 `## 二级标题` → 文件的 `# 一级标题`；原 `### 三级标题` → `### 三级标题`（保持锚点） |
| **编号保留** | 章节编号（如 5.1、5.2）在标题中保留，方便交叉引用 |

### 文件 frontmatter 模板

```markdown
---
outline: deep
---

# {章节标题}

{章节内容...}
```

### 拆分检查清单

- [ ] 每个方向一个文件（`directions/01-xxx.md` 到 `directions/0N-xxx.md`）
- [ ] 总览章节拆到 `overview/` 目录
- [ ] 生态章节拆到 `ecosystem/` 目录
- [ ] 分析章节拆到 `analysis/` 目录
- [ ] 附录保留为单文件 `appendix.md`
- [ ] 所有内部链接更新为相对路径
- [ ] 图片和静态资源移到 `public/`

### 4.2 可排序表格 (theme/index.ts)

在主题扩展中实现客户端表格排序：

```typescript
import DefaultTheme from 'vitepress/theme'
import type { Theme } from 'vitepress'
import './custom.css'

export default {
  extends: DefaultTheme,
  enhanceApp({ router }) {
    if (typeof window !== 'undefined') {
      const initSortableTables = () => {
        document.querySelectorAll('.vp-doc table').forEach((table) => {
          const thead = table.querySelector('thead')
          if (!thead || table.classList.contains('sortable-init')) return
          table.classList.add('sortable-init')
          const ths = thead.querySelectorAll('th')
          ths.forEach((th, colIdx) => {
            th.style.cursor = 'pointer'
            th.style.userSelect = 'none'
            th.title = '点击排序'
            const indicator = document.createElement('span')
            indicator.className = 'sort-indicator'
            indicator.textContent = ' ⇅'
            indicator.style.opacity = '0.4'
            indicator.style.fontSize = '0.8em'
            th.appendChild(indicator)

            let asc = true
            th.addEventListener('click', () => {
              const tbody = table.querySelector('tbody')
              if (!tbody) return
              const rows = Array.from(tbody.querySelectorAll('tr'))
              rows.sort((a, b) => {
                const aText = a.children[colIdx]?.textContent?.trim() ?? ''
                const bText = b.children[colIdx]?.textContent?.trim() ?? ''
                const aNum = parseFloat(aText.replace(/[^\d.\-]/g, ''))
                const bNum = parseFloat(bText.replace(/[^\d.\-]/g, ''))
                if (!isNaN(aNum) && !isNaN(bNum)) {
                  return asc ? aNum - bNum : bNum - aNum
                }
                return asc
                  ? aText.localeCompare(bText, 'zh')
                  : bText.localeCompare(aText, 'zh')
              })
              rows.forEach((row) => tbody.appendChild(row))
              ths.forEach((h) => {
                const ind = h.querySelector('.sort-indicator')
                if (ind) { ind.textContent = ' ⇅'; ind.style.opacity = '0.4' }
              })
              indicator.textContent = asc ? ' ↑' : ' ↓'
              indicator.style.opacity = '1'
              asc = !asc
            })
          })
        })
      }

      router.onAfterRouteChanged = () => setTimeout(initSortableTables, 100)
      if (document.readyState === 'complete') {
        setTimeout(initSortableTables, 100)
      } else {
        window.addEventListener('load', () => setTimeout(initSortableTables, 100))
      }
    }
  },
} satisfies Theme
```

**功能**：
- 点击任意表头即可排序（升序/降序切换）
- 自动检测数字列（数字排序）vs 文本列（中文 locale 排序）
- 排序指示器 ⇅ / ↑ / ↓
- 页面切换后自动重新初始化

## 第五轮：部署上线

### 5.1 GitHub Actions 自动部署

```yaml
name: Deploy to GitHub Pages

on:
  push:
    branches: [main]
  workflow_dispatch:

permissions:
  contents: read
  pages: write
  id-token: write

concurrency:
  group: pages
  cancel-in-progress: false

jobs:
  build:
    runs-on: ubuntu-latest
    steps:
      - uses: actions/checkout@v4
        with:
          fetch-depth: 0

      - uses: actions/setup-node@v4
        with:
          node-version: 20
          cache: npm

      - run: npm ci

      - name: Generate full report for download
        run: npm run merge

      - name: Copy full report to public
        run: cp {合并输出文件名}.md docs/public/

      - name: Build VitePress
        run: npm run docs:build

      - uses: actions/configure-pages@v4

      - uses: actions/upload-pages-artifact@v3
        with:
          path: docs/.vitepress/dist

  deploy:
    environment:
      name: github-pages
      url: ${{ steps.deployment.outputs.page_url }}
    needs: build
    runs-on: ubuntu-latest
    steps:
      - name: Deploy to GitHub Pages
        id: deployment
        uses: actions/deploy-pages@v4
```

**关键设计**：
- CI 中先运行 `merge` 脚本生成完整版 → 复制到 `public/` → 再构建
- 这样完整版 Markdown 文件可直接通过 URL 下载

### 5.2 合并脚本 (scripts/merge.sh)

```bash
#!/bin/bash
# 将多个 VitePress 内容页面合并为单文件完整版报告
set -e
cd "$(dirname "$0")/.."

OUTPUT="{输出文件名}.md"

# 写入报告头
cat > "$OUTPUT" << 'HEADER'
# {报告标题}

> **版本**：v1.0 · **更新日期**：{日期}
> **覆盖范围**：{范围} · **收录论文**：~{N} 篇

---

HEADER

# 按报告顺序定义页面列表
PAGES=(
  "docs/overview/executive-summary.md"
  "docs/overview/definition.md"
  "docs/overview/timeline.md"
  # ... 所有方向文件
  "docs/ecosystem/enterprises.md"
  "docs/ecosystem/opensource.md"
  "docs/analysis/tech-comparison.md"
  "docs/analysis/benchmarks.md"
  "docs/analysis/trends.md"
  "docs/analysis/conclusion.md"
  "docs/appendix.md"
)

for page in "${PAGES[@]}"; do
  if [[ -f "$page" ]]; then
    # 去掉 frontmatter (--- ... ---) 然后追加
    sed '/^---$/,/^---$/d' "$page" >> "$OUTPUT"
    echo -e "\n---\n" >> "$OUTPUT"
  else
    echo "WARNING: $page not found, skipping"
  fi
done

LINES=$(wc -l < "$OUTPUT")
echo "✅ 合并完成: $OUTPUT ($LINES 行)"
```

## 内容优化协议

### 事实审校流程

当需要对已发布网站内容做深度审校时：

```
1. 拆分审校任务为独立子任务（如：视频生成格局、具身AI深挖、垂直领域等）
2. 用 subagent 并行执行多个研究任务
3. 汇总 subagent 返回的研究数据
4. 按文件逐一修改（replace_string_in_file / multi_replace_string_in_file）
5. 修改后立刻 npm run docs:build 验证构建
6. git add + commit（详细中文 commit message）+ push
7. CI 自动部署
```

### 统计数据维护

每次内容变更后需更新以下位置的统计数字：

| 统计项 | 涉及文件 |
|--------|---------|
| 论文数 | `config.ts` description、`index.md` tagline、`conclusion.md`、`README.md`、`merge.sh` |
| 机构数 | `index.md` tagline、`conclusion.md`、`README.md` |
| 基准数 | `conclusion.md` |
| 方向数 | 同上 |

> ⚠️ 不要凭记忆更新——每次用 `grep_search` 搜索旧数字，确保全部更新到位。

### 章节编号维护

新增子章节时注意：
- 后续章节编号需手动递增（VitePress 不自动编号）
- 小结表格（方向小结）通常是文件最后一个 `###`，新增内容插入其前面
- 用 `grep_search` 搜索 `### X.Y` 确认编号连续

## 常见陷阱与应对

| 陷阱 | 应对策略 |
|------|---------|
| `base` 路径错误导致 404 | `config.ts` 的 `base` 必须与 GitHub 仓库名一致，包括前后的 `/` |
| favicon 不显示 | `head` 中的 href 必须包含 `base` 前缀（如 `/world-model-survey/favicon.svg`） |
| 公式不渲染 | 确保 `markdown: { math: true }` 已设置；安装 `markdown-it-mathjax3` |
| 表格在移动端溢出 | `custom.css` 中设置 `display: block; overflow-x: auto; white-space: nowrap` |
| 搜索不支持中文 | VitePress 内置 local search 已支持中文，确保 `lang: 'zh-CN'` |
| 完整版下载链接 404 | CI 中必须先 merge → 复制到 public → 再 build |
| 排序功能页面切换后失效 | `router.onAfterRouteChanged` 钩子重新初始化 |
| 编辑后构建失败 | 每次编辑后立即 `npm run docs:build` 验证 |
| Markdown 中 `\|` 导致表格解析错误 | 公式中的 `|` 用 `\vert` 替代 |
| 大量表格导致页面慢 | 可排序初始化用 `setTimeout(fn, 100)` 延迟，避免阻塞渲染 |

| 项目初始化 | `create_file` + `run_in_terminal` | 创建脚手架 + npm install |
| 内容拆分 | `read_file` + `create_file` | 读取单文件 → 拆分创建多文件 |
| 配置编写 | `create_file` | 一次性写入 config.ts、theme/*.ts、*.css |
| 内容修改 | `replace_string_in_file` / `multi_replace_string_in_file` | 精准编辑已有内容 |
| 构建验证 | `run_in_terminal`（`npm run docs:build`） | 每次编辑后验证 |
| 部署 | `run_in_terminal`（`git add + commit + push`） | 触发 CI 自动部署 |
| 本地预览 | `run_in_terminal`（`npm run docs:dev`）+ `open_simple_browser` | 开发时实时预览 |
| 统计审计 | `grep_search` + `runSubagent` | 搜索数字 → subagent 精确计数 |
| 链接验证 | `fetch_webpage` | 批量验证 arXiv/GitHub 链接可达性 |

## 质量标准

### 必须达到

- [ ] `npm run docs:build` 零错误
- [ ] 所有页面有 `outline: deep` frontmatter
- [ ] 导航栏 + 侧边栏覆盖所有页面
- [ ] 本地搜索可用（`search.provider: 'local'`）
- [ ] KaTeX 公式渲染正常
- [ ] 表格可点击排序
- [ ] 移动端表格可水平滚动
- [ ] favicon 在标签页显示
- [ ] GitHub Actions 自动部署成功
- [ ] 完整版 Markdown 可下载
- [ ] `lastUpdated` 时间戳显示正确
- [ ] 所有 arXiv/GitHub 链接有特殊样式（虚线下划线）

### 建议达到

- [ ] 首页 Feature 卡片覆盖所有研究方向
- [ ] 深色模式下品牌色正确
- [ ] README.md 与网站 tagline 统计数据一致
- [ ] merge.sh 输出与网站内容同步
- [ ] SVG 时间线等可视化资源放在 `public/`
